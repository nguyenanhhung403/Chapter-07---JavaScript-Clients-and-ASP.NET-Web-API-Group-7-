A. The Edge Layer

  • CLIENT User: Browser or Mobile App sending HTTP Requests (GET/POST).
  • PROXY Reverse Proxy (Nginx/IIS): The first line of defense. Handles SSL termination, load balancing, and forwarding requests to Kestrel.

B. Web Server & Host

Kestrel Server: The cross-platform, high-performance web server implementation for ASP.NET Core. It receives the request from the proxy, constructs the HttpContext, and pushes it into the Middleware Pipeline.

C. Middleware Pipeline

"The Heart of ASP.NET Core - Order is critical"

  • Exception Handling: Catches errors globally to ensure a graceful failure response.
  • HSTS / HTTPS: Enforces security, ensuring all communication uses SSL/TLS.
  • Routing: Determines which endpoint (Controller) matches the request URL.
  • Authentication: Answers "Who are you?" identifying the user.

D. Application Logic

  • Controller
    Accepts the request, performs Model Validation, and delegates to services.
  • Business Services
    Encapsulates complex business rules, calculations, and external integrations (e.g., Email).
  • Data Access (EF Core)
    Interacts with the SQL Server database to Create, Read, Update, or Delete data.
